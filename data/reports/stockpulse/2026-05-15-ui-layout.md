## Top 3 user-visible issues

1. **F1 / R1 — `WatchlistGrid` grid columns key off viewport, not the available content area.**
   At 1024 px, cards flip size by ~36% across the breakpoint with no proportional reason. `grid-cols-1 sm:grid-cols-2 md:grid-cols-3 lg:grid-cols-4 xl:grid-cols-5` was set at viewport widths but `AppLayout` has no sidebar, so the content area IS the viewport — meaning the breakpoint set should match content density, not Tailwind's defaults. (Fix: `WatchlistGrid.tsx` → use `auto-fill` minmax grid template.)
2. **F2 / R2 — `TickerDetailModal` wastes ~50% of 1440 px and breaks at 320 px.**
   Fixed `max-w-2xl` + rigid 3-col stats grid. On large displays the modal is a column of empty space; on mobile the 3-col stats overflow horizontally. (Fix: responsive `max-w-`, stats grid reflows to 2-col at md and 1-col at sm.)
3. **R7 — `TickerCard` remove button is hover-only with ~20 px hit area.**
   Hidden behind `opacity-0 group-hover:opacity-100` with no touch alternative; size below WCAG 2.5.5 minimum (24 px). On touch devices the button is fully unreachable. (Fix: always visible with reduced opacity, ≥44 px tap target.)

## UNFIXED — needs human decision

- **F4 — `StatusBar` height.** The component baselines at 36 px on desktop but feels heavy on mobile. Reducing it changes vertical rhythm across the whole app. Pick: keep current 36 px / reduce to 28 px on `sm:` / make it auto-height with `py-1.5`. Label-binding portion of the fix IS written (replaces the hardcoded "5 s poll" with the actual poll interval from config).
- **S10 — `TopBar` grouping.** Logo + nav + connection indicator + theme toggle are currently flat-spaced; tightening to a `gap-6` left-group / `gap-3` right-group split would help, but choice of groupings depends on which actions are primary. Defer to your IA call.

## Cross-cutting notes from auditors

- **AppLayout is vertical** (TopBar + main + StatusBar) — no sidebar, so the audit's mobile vs. desktop story is mostly about the `main` column behavior, not chrome-eats-viewport.
- **Hardcoded hex in `IntradayChart.tsx`** is flagged by Code Quality FE review but is a color-system concern, not spatial. Deferred — should be picked up by a future styling pass.
- **Viewport meta tag present and correct** in `client/index.html`.
- **Touch targets in forms** (`AlertEditor`, `SymbolSearch`) are the most consistent responsive risk — `py-1` produces ~32 px tap height, just below WCAG. R4/S4 raise to `py-2`.
